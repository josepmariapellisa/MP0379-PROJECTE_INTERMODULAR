    Documentació Tècnica: Eina d'Auditoria de Xarxa amb Docker

    Projecte Intermodular - 2n ASIX
    Data de generació: [Data actual]

    1. Introducció

    Aquest document descriu el desenvolupament i l'ús d'una eina d'auditoria de xarxa implementada mitjançant Docker. L'objectiu és disposar d'un entorn portable i reproducible per a realitzar tasques d'auditoria de seguretat i reconeixement de xarxes, empaquetat en una imatge Docker que pot ser executada des d'un pendrive.

    2. Dockerfile i Configuració de la Imatge

    El punt de partida és el fitxer Dockerfile, que defineix la imatge base i les eines necessàries per a l'auditoria.

    2.1 Contingut del Dockerfile

    FROM python:3.9-slim
ENV DEBIAN_FRONTEND=noninteractive
RUN apt-get update && apt-get install -y \
    build-essential \
    libxml2-dev \
    libxslt1-dev \
    && rm -rf /var/lib/apt/lists/*
WORKDIR /app
COPY requirements.txt /app/
RUN pip install --no-cache-dir -r requirements.txt
RUN pip install theHarvester
COPY . /app
RUN mkdir -p /app/dades
ENTRYPOINT ["python", "main.py"]

    2.2 Explicació

    • Imatge base: Python 3.9 slim.
    • Instal·lació de dependències: eines de compilació i biblioteques per a l'anàlisi XML/XSLT.
    • Instal·lació de Python: requirements.txt i l'eina theHarvester.
    • Directori de treball: /app.
    • Punt d'entrada: executa main.py quan s'inicia el contenidor.

    Per portabilitat, s'exporta la imatge a un fitxer comprimit:

    docker save -o auditoria_pen2.tar auditoria_pen2

    4. Execució de l'Auditoria des d'un Pendrive

    El procés està dissenyat per ser executat des d'un Live USB o un pendrive amb el fitxer de la imatge.

    4.1 Comprovació del pendrive muntat

    ls /media/ubuntu/PENDRIVE/

    4.2 Càrrega de la imatge Docker al sistema Live

    docker load < /media/ubuntu/PENDRIVE/auditoria_pendrive.tar

    4.3 Execució de l'auditoria amb volum per als resultats

    docker run -it --rm -v /media/ubuntu/PENDRIVE/resultats:/app/dades auditoria_pendrive

    Explicació:
    - -it: mode interactiu.
    - --rm: elimina el contenidor després de l'execució.
    - -v: munta el directori del pendrive dins el contenidor per guardar resultats.

    (Espai per captura del procés d'execució i la sortida de l'auditoria)

    5. Configuració per a Clients amb Interfície Gràfica

    Si l'eina requereix sortida gràfica (per exemple, amb eines com chromium), es configura l'accés a X11:

    5.1 Permetre connexions locals de Docker a X11

    xhost +local:docker

    5.2 Execució amb suport gràfic

    docker run -it --rm \
  -e DISPLAY=$DISPLAY \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  -v /media/ubuntu/PENDRIVE/results:/app/dades \
  auditoria_pendrive

    Explicació:
    - -e DISPLAY: passa la variable d'entorn per a la interfície gràfica.
    - -v /tmp/.X11-unix: comparteix el socket de X11 amb el contenidor.

    (Espai per captura de l'execució amb sortida gràfica)

└── results/ (directori per als resultats de l'auditoria)

    7. Conclusió

    Aquesta eina d'auditoria empaquetada amb Docker ofereix una solució portable, reproducible i aïllada per a tasques de reconeixement de xarxa i seguretat informàtica. Gràcies a la containerització, es pot executar en qualsevol sistema amb Docker instal·lat sense conflictes de dependències.
