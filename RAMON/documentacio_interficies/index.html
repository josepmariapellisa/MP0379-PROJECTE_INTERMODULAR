    Documentació: Interfície i Escanejos Nmap

    Aquest document. Se centra en tres pilars: la gestió dels subprocessos nmap, l'estructura de la interfície gràfica (GUI) amb Tkinter, i el mètode d'execució en fils (threading) que permet a l'aplicació ser responsiva i cancel·lable.

    1. Gestió de Processos (Nmap)

    Per poder executar escanejos nmap (que poden trigar molt) sense bloquejar el programa i permetent a l'usuari aturar-los, s'utilitza una funció wrapper anomenada run_nmap_stoppable.

    • Execució no bloquejant: Fa servir subprocess.Popen per llançar nmap com un procés separat.
    • Variable Global current_process: Emmagatzema el procés en execució en una variable global perquè altres funcions (com la d'aturar) hi puguin accedir.
    • Bucle de Comprovació: Entra en un bucle while True que comprova dues coses:
      1. stop_event.is_set(): Si l'usuari ha demanat aturar l'escaneig. Si és així, termina el procés (.terminate()) i retorna.
      2. current_process.poll(): Si el procés ha acabat per si mateix. Si és així, recull la sortida (.communicate()) i la retorna.
    • Gestió d'Errors: Captura excepcions, com ara si nmap no està instal·lat.
    # FUNCIO PER ATURAR PROCESSOS
current_process = None
def run_nmap_stoppable(args_list, stop_event):
    global current_process
    cmd = ["nmap"] + args_list
    try:
        current_process = subprocess.Popen(cmd, stdout=subprocess.PIPE, stderr=subprocess.PIPE, text=True)
        # Esperar que acabi o que es demani aturada
        while True:
            if stop_event.is_set():
                current_process.terminate()
                current_process.wait()
                return "", "Escaneig aturat per l'usuari", -1

            returncode = current_process.poll()
            if returncode is not None:
                stdout, stderr = current_process.communicate()
                return stdout, stderr, returncode
            time.sleep(0.1)
    except Exception as e:
        return "", f"Error: {e}", 1
    finally:
        current_process = None

    2. Funcions d'Escaneig (Nmap)

    Les funcions ping_scan, port_scan i version_scan són essencialment "wrappers" (embolcalls) que defineixen els arguments específics de nmap per a cada tipus d'escaneig. Totes elles deleguen l'execució real a run_nmap_stoppable, passant-li els arguments correctes i el stop_event.

    • ping_scan: Utilitza -sn (Scan de Ping, sense escaneig de ports).
    • port_scan: Utilitza -sT --top-ports 100 (Escaneig TCP Connect als 100 ports més comuns).
    • version_scan: Utilitza -sV --top-ports 100 (Detecció de Versió de Serveis als 100 ports més comuns).
    def ping_scan(target, stop_event=None):
    """Escaneig de ping bàsic"""
    if stop_event:
        stdout, stderr, rc = run_nmap_stoppable([target, "-sn"], stop_event)
    else:
        return f"Error executant ping scan:"
    return stdout or stderr

def port_scan(target, stop_event=None):
    """Escaneig de ports bàsic"""
    if stop_event:
        stdout, stderr, rc = run_nmap_stoppable([target, "-sT", "--top-ports", "100"], stop_event)
    else:
        return f"Error executant port scan:"
    return stdout or stderr

def version_scan(target, stop_event=None):
    """Escaneig de versió"""
    if stop_event:
        stdout, stderr, rc = run_nmap_stoppable([target, "-sV", "--top-ports", "100"], stop_event)
    else:
        return f"Error executant version scan:"
    return stdout or stderr

    • __init__: El constructor defineix tots els elements visuals (widgets).
    • frm_top: Un Frame superior que conté l'etiqueta (Label) i el camp d'entrada (Entry) per a l'objectiu (target).
    • btn_frame: Un Frame que agrupa tots els botons d'acció (Button) per a una millor organització.
    • Àrea de Resultats: Un Frame inferior que conté el widget principal Text (on es mostra la sortida) i una Scrollbar (barra de desplaçament) associada.

    4. Gestió d'Execució i Fils (Threading)

    Aquesta és la part més important per a la interactivitat de la GUI. Si es cridés a nmap directament des del botó, la interfície es "congelaria" fins que acabés l'escaneig. Per evitar-ho, cada escaneig es llança en un fil (thread) separat.

    • execucions(self, func, *args): Aquest mètode és el cor del sistema.
      1. Rep una funció (func) i els seus arguments (*args).
      2. Defineix una funció interna task().
      3. Crea un nou threading.Thread que executarà task().
      4. El fil es marca com a daemon=True perquè es tanqui automàticament si es tanca el programa principal.
      5. Inicia el fil (.start()).
    • task(): Aquesta funció (ja dins del nou fil) executa la funció d'escaneig real (func(*args)), recull la sortida i, un cop acabada, l'afegeix a l'àrea de text (self.append_text).
    • start_... Mètodes: Els mètodes com start_ping_scan actuen com a "cola":
      1. Comproven si ja hi ha un escaneig en curs.
      2. Preparen la GUI (_prepare_scan).
      3. Obtenen l'objectiu del camp d'entrada.
      4. Criden a self.execucions per llançar l'escaneig en un fil.
    • append_text: Aquesta funció de suport afegeix text a l'àrea de resultats i, crucialment, crida a self.root.update() per forçar la GUI a refrescar-se immediatament, fins i tot si el fil principal està ocupat.
        # --- EXECUCIÓ I RESULTATS ---
    def execucions(self, func, *args):
        def task():
            try:
                output = func(*args)
                self.scan_in_progress = False

                if self.stop_event.is_set():
                    self.append_text("[!] ⏹ Escaneig ATURAT per l'usuari")
                elif output:
                    self.append_text(output)
                    self.append_text("[✓] Escaneig COMPLETAT")
                else:
                    self.append_text("[!] Sense sortida o error desconegut.")
            except Exception as e:
                self.scan_in_progress = False
                self.append_text(f"[ERROR] {e}")

        t = threading.Thread(target=task)
        t.daemon = True
        t.start()

        def start_ping_scan(self):
        """Inicia l'escaneig de ping"""
        if self.scan_in_progress:
            messagebox.showwarning("Escaneig en curs", "Ja hi ha un escaneig en curs. Atura'l primer.")
            return
        self._prepare_scan()
        target = self.entry_target.get().strip()
        if not target:
            messagebox.showerror("Error", "Indica una xarxa/host al camp.")
            return
        self.execucions(self._do_ping_scan, target)

    def _do_ping_scan(self, target):
        self.append_text(f"[*] Iniciant Ping scan: {target}")
        self.append_text("[!] Prem 'Atura Escaneig' per cancel·lar")
        out = ping_scan(target, self.stop_event)
        return out

    5. Mecanisme d'Aturada

    El botó "ATURA" permet a l'usuari cancel·lar un escaneig en curs.

    • peticio_stop_auditoria(self): Aquesta funció és cridada pel botó "⏹ ATURA".
      1. Comprova si self.scan_in_progress és cert.
      2. Activa l'esdeveniment: self.stop_event.set(). Aquest és el senyal que el bucle dins de run_nmap_stoppable està escoltant.
      3. Per a una aturada més immediata, accedeix a la variable global current_process i crida a .terminate() directament, matant el procés nmap a l'instant.
        def peticio_stop_auditoria(self):
        if self.scan_in_progress:
            self.append_text("[!] Sol·licitant aturada...")
            self.stop_event.set()
            global current_process
            if current_process:
                try:
                    current_process.terminate()
                except:
                    pass
        else:
            self.append_text("[!] No hi ha cap escaneig en curs")

    6. Escaneig de Vulnerabilitats (NSE)

    S'ha incorporat una funció específica vuln_scan per a l'auditoria de seguretat avançada. Aquesta funcionalitat aprofita la potència del Nmap Scripting Engine (NSE) per detectar CVEs i fallades de seguretat conegudes en els serveis identificats.

    Arguments Clau

    • -sV: Detecta la versió del servei. Nota: Aquest argument és imprescindible perquè els scripts de vulnerabilitat sàpiguen exactament què buscar basant-se en la versió del programari.

    • --script vuln: Executa una col·lecció completa d'scripts classificats dins la categoria "vulnerabilitat" contra l'objectiu.

    Tractament de la Sortida

    A diferència dels escanejos simples, aquest mòdul inclou un processament visual per facilitar l'anàlisi:

    • Funció de neteja (prettify_vuln): Processa el text en brut per eliminar soroll.
    • Ressaltat a la GUI: La interfície analitza el text en temps real buscant paraules clau crítiques com "VULNERABLE" o "CVE-". Quan les troba, les ressalta visualment en vermell (etiqueta VULN_HIGHLIGHT) per cridar l'atenció de l'auditor.
    # Funció d'escaneig principal
def vuln_scan(target, stop_event=None):
    if stop_event:
        # --script vuln: Executa scripts de detecció de vulnerabilitats
        # -sV: Necessari per saber la versió del servei
        cmd = ["nmap", target, "-sV", "--script", "vuln"]
        stdout, stderr, rc = run_stoppable_command(cmd, stop_event)

        # Si triga molt i l'usuari cancel·la
        if rc == -1: return "Escaneig de vulnerabilitats aturat."

        return prettify_vuln(stdout + "\n" + stderr)
    else: return "Error intern"

# Helper per netejar text ANSI i detectar fallades crítiques
def prettify_vuln(raw: str) -> str:
    lines = raw.splitlines()
    out_lines = []
    for raw_line in lines:
        line = strip_ansi(raw_line).rstrip()
        # Marca específica per injeccions SSL
        if "|_ssl-ccs-injection: VULNERABLE" in line:
             out_lines.append(f"TAG_FAIL::{line}")
        else:
             out_lines.append(line)
    return "\n".join(out_lines)
